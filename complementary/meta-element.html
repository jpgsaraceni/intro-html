Meta Elements

<meta> elements are used to represent metadata that cannot be represented by other metadata elements. Asides the <meta> element, other metadata elements are <base>, <command> (depracated), <link>, <noscript>, <script> and <title>.

Because they represent metadata content, that is not rendered on screen, they are always empty elements.

Most meta elements are always direct children of the <head> element, though there are some exceptions.

Types of metadata

The attributes define the type of metadata of the element.

  • Document-Level Metadata:

    applies to the whole page. This type of metadata contains a name attribute and a content attribute, which define a name-value pair of metadata for the page. See here (MDN) for more details about standard metadata names.

  • Pragma Directive:

    http-equiv and content attribute pairs define HTTP headers equivalents. For more information, see MDN.

  • Charset Declaration:

    character encoding for the document. Defined by the charset attribute. This element must be entirely within the first 1024 bytes of the document. UTF-8 is the only valid encoding for HTML5.

  • User-Defined Metadata:

    defined by the itemprop attribute, adds properties to the element. For more information, see MDN.
